**内容策略**

Tagline公式：
[解决什么问题] for [目标用户]

我的：
"API uptime monitoring for indie developers"

描述结构：
1. 痛点（为什么需要）
2. 解决方案（怎么解决）
3. 核心功能（3个关键特性）
4. 差异化（为什么选我们）

视频脚本：
0-10s: 痛点场景
10-60s: 产品demo
60-90s: Call to action
