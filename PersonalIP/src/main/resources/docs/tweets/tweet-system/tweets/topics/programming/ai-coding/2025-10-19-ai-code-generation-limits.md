# AI生成的代码，我不会直接用的5个原因

## 推文内容

AI编程工具很强
但盲目信任 = 踩坑 🕳️

**AI生成代码的问题：**

**1. 上下文缺失**

AI看到的：
你粘贴的几十行代码

看不到的：
- 项目整体架构
- 业务逻辑
- 历史决策

生成的代码：
可能和项目风格不一致

**2. 过时的知识**

ChatGPT训练数据：
截止2023年

现在是2024年：
- 框架新版本
- API变更
- 最佳实践更新

它可能给你过时的方案

**3. 安全隐患**

示例：AI生成的SQL查询

```java
// AI生成
String sql = "SELECT * FROM users WHERE name = '" + name + "'";

// SQL注入漏洞！
```

它不会主动考虑安全

**4. 性能问题**

AI倾向于：
- 简单粗暴的实现
- 不考虑大数据量
- 忽视性能优化

示例：
```java
// AI可能生成
for (User user : users) {
    db.getOrders(user.getId());  // N+1问题
}

// 应该
List<Order> orders = db.getOrdersByUserIds(userIds);  // 批量查询
```

**5. 过度工程**

问：实现用户登录
AI：给你完整的OAuth2+JWT+Refresh Token+多因素认证

你需要的：
简单的username/password

**我的AI代码使用原则：**

**1. AI生成→人工审查**

Checklist:
□ 逻辑正确？
□ 安全性？
□ 性能？
□ 符合项目规范？
□ 测试覆盖？

全过了才用

**2. AI作为起点，不是终点**

Workflow:
1. AI生成框架代码
2. 我调整业务逻辑
3. 优化性能
4. 添加错误处理
5. 写测试
6. 完善文档

**3. 简单任务放心用**

适合AI：
✓ Boilerplate代码
✓ 单元测试
✓ 简单CRUD
✓ 类型转换
✓ 配置文件

不适合AI：
✗ 核心业务逻辑
✗ 复杂算法
✗ 安全关键代码
✗ 性能敏感部分

**4. 验证再验证**

不要：
复制→粘贴→提交

要做：
复制→理解→测试→优化→提交

**5. 学习AI的思路**

不是：
拿代码就跑

而是：
看它怎么解决问题
学习思路
举一反三

**真实案例：**

任务：实现文件上传

AI生成代码：
- 直接存内存
- 没限制文件大小
- 没验证文件类型
- 没处理并发

能用吗？能
生产环境敢用吗？不敢

我的修改：
+ 流式处理
+ 文件大小限制
+ 类型白名单
+ 临时文件清理
+ 错误处理
+ 安全检查

20行→80行
但靠谱了

**AI时代的开发者**

不是：
被AI取代

而是：
会用AI的开发者
取代不会用AI的开发者

但"会用"≠"盲目用"

会用 = 知道何时用、如何用、如何验证

**给新手的建议：**

别让AI阻碍学习

用AI前：
先自己思考
尝试自己实现
卡住了再问AI

用AI后：
理解它的代码
问为什么这样写
学习新知识

AI是助手
不是老师

记住：
AI生成代码的速度很快
但修Bug的时间可能更长

质量 > 速度
