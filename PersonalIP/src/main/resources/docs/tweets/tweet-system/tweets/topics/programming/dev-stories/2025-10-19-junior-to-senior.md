# 从Junior到Senior：5年我学到的10件事

## 推文内容

5年前：刚毕业的菜鸟
现在：带团队的Senior

分享最重要的10个认知 🚀

**1. 代码质量 > 代码数量**

Junior我：
一天写1000行代码
觉得自己很牛

Senior我：
一天重构删500行代码
系统更稳定

Less is more

**2. 解决问题 > 炫技**

Junior：用最新最酷的技术
Senior：用最合适的技术

不是不学新技术
是要知道when to use

**3. 沟通能力 = 技术能力**

写代码只占工作的40%
剩下60%：
- 需求沟通
- Code Review
- 技术方案讨论
- 跨团队协作

不会沟通=能力打折

**4. 理解业务很重要**

Junior：
PM说啥做啥
不关心为什么

Senior：
理解业务目标
提供更好方案

技术服务业务
不是为了技术而技术

**5. 预防问题 > 解决问题**

Junior：
写代码快
出Bug也快

Senior：
写代码慢一点
但考虑边界情况
提前防御

**6. 文档和测试不是浪费时间**

Junior：
觉得写文档没用
不写测试

Senior：
文档救过我无数次
测试避免了无数Bug

**7. 承认不知道不丢人**

Junior：
装懂
怕被看不起

Senior：
不懂就问
快速学习

没人什么都懂

**8. 代码是给人看的**

Junior：
写给机器的代码

Senior：
写给6个月后的自己
写给团队其他人

可维护性 > 炫技

**9. 提前规划 > 返工**

Junior：
先写代码
遇到问题再改

Senior：
先想清楚
画个设计图
再动手

磨刀不误砍柴工

**10. 技术债要还**

Junior：
"先实现功能，以后再优化"
（以后从不来临）

Senior：
定期重构
不让债务累积

**最大的转变：**

Junior思维：
我的代码能跑就行

Senior思维：
- 能跑 ✓
- 可维护 ✓
- 可扩展 ✓
- 性能好 ✓
- 有测试 ✓
- 有文档 ✓

**技术之外：**

Junior关心：
- 用什么框架
- 学什么技术
- 写多少代码

Senior关心：
- 解决什么问题
- 创造什么价值
- 如何帮助团队

**给Junior开发的建议：**

1. 多问why
   不只是how

2. 读优秀代码
   GitHub上的大项目

3. 写博客
   教是最好的学

4. 贡献开源
   学习协作

5. 找Mentor
   少走弯路

**从Junior到Senior**
不只是技术成长
更是思维转变

需要：
- 时间
- 犯错
- 反思
- 学习

但绝对值得 💪

你在哪个阶段？
