# 产品验证：我如何用$0成本验证SaaS想法

## 推文内容

上个月花了0美元
验证了3个SaaS想法
其中1个已经有5个付费意向

分享我的具体方法

**第1步：痛点验证（Day 1-3）**

去Reddit/Twitter找抱怨：
- r/SaaS
- r/Entrepreneur
- IndieHackers

搜索关键词：
"frustrated with..."
"hate using..."
"wish there was..."

找到12条相关抱怨
选了3个最频繁的痛点

**第2步：竞品分析（Day 4-5）**

每个想法找3-5个竞品：
- 定价多少？
- 用户评价怎样？
- 缺失什么功能？

工具：
- SimilarWeb (流量估算)
- BuiltWith (技术栈)
- Product Hunt (用户反馈)

发现机会：
现有产品太复杂
定价$99+/月
中小企业负担不起

**第3步：Landing Page测试（Day 6-7）**

用Carrd搭建落地页：
- 清晰价值主张
- 核心3个功能
- Waitlist表单

成本：$19/年

文案公式：
问题 + 解决方案 + 社会证明

**第4步：流量获取（Day 8-14）**

免费渠道：
1. Twitter发帖（带#标签）
2. Reddit相关subreddit
3. Facebook groups
4. LinkedIn个人动态

每个想法发10-15个帖子

**结果数据：**

想法A（项目管理工具）：
- 访问：143人
- Waitlist：8人（5.6%）
- 付费意向：0

想法B（邮件模板SaaS）：
- 访问：89人
- Waitlist：3人（3.4%）
- 付费意向：0

想法C（API监控工具）：
- 访问：267人
- Waitlist：34人（12.7%）
- 付费意向：5人
- 愿付价格：$29-49/月

**关键洞察：**

1. 转化率>10%是好信号
2. 有人愿意预付费=强需求
3. 技术类产品开发者付费意愿高

**验证清单：**

继续开发的条件：
- Landing page转化率>8%
- 至少5个愿意预付的用户
- 竞品定价有空间
- 技术实现<30天

想法C全都满足
现在开始做MVP

总成本：$19
时间投入：14天（每天2小时）

不要盲目开发
先验证需求
