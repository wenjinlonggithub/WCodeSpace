## 推文内容 (中文版)

我用这个5步框架验证startup ideas：

1️⃣ **你的痛点测试**
自己是不是愿意付费解决这个问题？
→ 如果答案是no，停止

2️⃣ **10人验证**
找10个目标用户，问他们会不会付费
→ 至少3人说yes才继续

3️⃣ **定价测试**
报个价格，看他们反应
→ 太便宜 = 价值不够
→ "贵但我会考虑" = 找到了

4️⃣ **48小时MVP**
最小可行版本，功能丑但能用
→ 拿给那10个人试

5️⃣ **第一笔收入**
在写任何"真正的"代码之前拿到钱
→ 这才是真validation

我3个side projects，只有通过这个流程的那个成功了。

另外2个？死在step 2。

## Tweet Content (English Version)

My 5-step framework to validate startup ideas:

1️⃣ **Your Pain Test**
Would YOU pay to solve this problem?
→ If no, stop here

2️⃣ **10-Person Validation**
Find 10 target users, ask if they'd pay
→ Need at least 3 "yes" to proceed

3️⃣ **Pricing Test**
Name a price, watch their reaction
→ Too cheap = not valuable enough
→ "Expensive but I'd consider" = sweet spot

4️⃣ **48-Hour MVP**
Minimum viable product, ugly but functional
→ Show those 10 people

5️⃣ **First Revenue**
Get money BEFORE writing any "real" code
→ This is true validation

I've done 3 side projects. Only the one that passed this framework succeeded.

The other 2? Died at step 2.

## 补充内容准备

### 回复1: 常见错误
"我犯过的5个validation错误：

❌ 问亲朋好友（他们会撒谎）
❌ 问"你觉得这个idea怎么样"（毫无意义）
❌ 花3个月做MVP（太久）
❌ 免费给用户用（不是真validation）
❌ 相信"等功能完善了我就付费"

✅ 正确做法：
→ 找陌生的目标用户
→ 问"你会付$X解决这个问题吗"
→ 48小时做MVP
→ 先收钱
→ 相信行动，不相信承诺"

### 回复2: 如何找到10个目标用户
"我的方法：

1️⃣ **Reddit/论坛**
   找相关subreddit，看谁在抱怨这个问题

2️⃣ **LinkedIn**
   搜索目标职位，cold DM

3️⃣ **Twitter**
   搜索关键词，找正在经历痛点的人

4️⃣ **竞品用户**
   用类似产品的人可能不满意

5️⃣ **你的network**
   但只作为intro，不作为validation

关键：陌生人 > 熟人"

### 回复3: 48小时MVP例子
"我的一个48h MVP：

**问题**：开发者需要快速生成API文档

**48h MVP**：
→ 一个简单的web form
→ 贴API代码
→ 用GPT-4生成文档
→ 输出markdown

就这么简单。

没有：
❌ 用户系统
❌ 美观UI
❌ 高级功能

有：
✅ 核心价值
✅ 能演示
✅ 能收钱

第3天收到第一笔$29。"

### 回复4: 定价测试话术
"不要问：'你愿意付多少？'

要说：'这个服务$99/月，你会订阅吗？'

观察反应：

😱 '太贵了' → 价格高 or 价值低
🤔 '有点贵但有价值' → ✅ 甜蜜点
😊 '很便宜啊' → ⚠️ 定价太低

然后试探：
'如果$149呢？'

找到他们犹豫的点。"
